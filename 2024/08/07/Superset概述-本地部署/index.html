Superset

Welcome | Superset (apache.org)
https://github.com/apache/superset

— 简介 —

  1. Superset 是一个数据可视化和数据探索平台（数据可视化工具）
  2. Superset 可以取代或增强许多团队的专有商业智能工具
  3. Superset 与各种数据源很好地集成。

Superset 提供：

  • A no-code interface for building charts quickly

    用于快速构建图表的无代码界面

  • A powerful, web-based SQL Editor for advanced querying

    一个强大的、基于 Web 的 SQL 编辑器，用于高级查询

  • A lightweight semantic layer for quickly defining custom dimensions and metrics

    一个轻量级语义层，用于快速定义自定义维度和指标

  • Out of the box support for nearly any SQL database or data engine

    开箱即用，支持几乎所有 SQL 数据库或数据引擎

  • A wide array of beautiful visualizations to showcase your data, ranging from simple bar charts to geospatial visualizations

    各种精美的可视化效果来展示您的数据，范围从简单的条形图到地理空间可视化效果

  • Lightweight, configurable caching layer to help ease database load

    轻量级、可配置的缓存层，有助于减轻数据库负载

  • Highly extensible security roles and authentication options

    高度可扩展的安全角色和身份验证选项

  • An API for programmatic customization

    用于编程定制的 API

  • A cloud-native architecture designed from the ground up for scale

    从头开始设计的云原生架构，旨在实现规模化

— 安装 —

建筑 |超集 — Architecture | Superset (apache.org)

官网提供多种安装方式：

Kubernetes、PyPI、Docker Compose、Docker

在 Quickstart 中官方建议使用 Docker Compose 方式，但这种构建方式不被建议在生产环境中使用

可参考：

使用Docker Compose安装本地安装 | Superset (zhaoweilong.github.io)

要注意 superset 不直接支持 Windows 操作系统，需要使用虚拟机或 WSL + Docker Desktop

apache/superset - Docker Image | Docker Hub

apache-superset:2.1详细部署流程

< superset 本地vmware虚拟机 ubuntu 环境 PyPI 部署实现 >

目的

  • 项目需求，用于生产的数据分析平台，使用PyPI部署
  • 测试与简单调用可移步docker部署

环境

  • VMware® Workstation 16 Pro
  • ubuntu-20.04.6-desktop-amd64

具体步骤

一、OS环境配置

  1. 下载 iso 文件
    Index of /ubuntu-releases/ | 清华大学开源软件镜像站 | Tsinghua Open Source Mirror

  2. 打开VMware，点击/文件/新建虚拟机

  3. 选择典型，下一步

  4. 选择安装程序光盘映像文件，找到刚才下载的iso文件，下一步

  5. 填写信息（名称与登录密码），下一步

  6. 填写信息，找个合适的目录位置，下一步

  7. 20G（仅作项目环境调试不长期使用），单个文件（确保性能但影响不大），下一步

  8. 点击完成

  9. 当前页面，点击开启此虚拟机

  10. 等一会，出现下面页面，继续等待

  11. 鼠标移动到虚拟机窗口可操作，按 CTRL+ALT 快捷键回到主机，自动安装结束出现下面页面
    点击，输入先前设置的密码，回车，登入系统

二、Superset环境配置

  1. 进入系统，出现弹窗，全都关掉Skip/Next/Done

  2. 打开左上角火狐浏览器，搜索Superset

  3. 一般是搜索结果第一个，进入官网，点击 Get Started

  4. 点击左侧栏 /Installation/PyPI

  5. 右侧找到 ubuntu20.04 对应的依赖项下载命令复制

  6. 粘贴到终端（打开左侧Home文件夹，右键空白处，点击Open in Terminal打开终端）

  7. 在终端窗口使用快捷键 CTRL+SHIFT+V，复制先前拷贝的命令，粘贴后回车运行（中间需输入密码和Y确定），运行结果如下图

  8. 手册往下翻，安装python的虚拟环境venv

  9. 出现了一个WARNING，告诉我们需要添加环境变量$PATH（可以使用 echo $PATH命令查看当前环境变量）

  10. 使用 sudo vi /etc/profile 进入配置文件进行编辑

  11. 到最后一行，先点击 i 键进入编辑模式，写入export PATH=$PATH:/home/krisnile(替换为自己的用户名)/.local/bin/（如果麻烦就在目录里找到文件使用其他编辑器或自行安装vim等，注意保存）

  12. 输入source /etc/profile载入修改后的配置文件

  13. 输入virtualenv —version查看到命令已经存在

  14. 使用 virtualenv superset创建名为superset的python虚拟环境，并使用source命令激活环境

    或者可以使用其他方式构建虚拟环境（推荐conda），也可参考文档

关于其他操作系统配置（网络，快捷键，美化等）这里不再赘述，如有需要自行配置即可

三、Superset下载安装与初始化

  1. 在刚刚创建的虚拟环境内，输入pip install apache-superset命令正式下载superset，等待结束

  2. 结束后，输入superset db upgrade命令进行初始化

    发现报错

  3. 检查发现superset2.x版本不再受支持
    Superset 2.x： ‘AttributeError： 模块’flask.json’没有属性’JSONEncoder’’ ·apache/超集 ·讨论 #27211 — Superset 2.x: AttributeError: module 'flask.json' has no attribute 'JSONEncoder' · apache/superset · Discussion #27211 (github.com)

如果更换版本成本太大，尝试了pip python和superset的升级也有一些问题

  1. 还是先根据提示先pip install —upgrade pip

再次运行superset db upgrade命令可能会出现 TypeError: init() got an unexpected keyword argument ‘unbound_message’是apache-superset 2.1的bug，需要降级flask，使用下面的命令：

python -m pip uninstall -y Flask

python -m pip install Flask==2.0.3

  1. 发现在终端输入superset是有输出的，所以还是配置问题

  2. 要设置一个FLASK_APP的环境变量，终端输入export FLASK_APP=superset

    再次运行superset db upgrade

  1. 看文档要进行初始化配置 翻译：注意，对于Superset的生产实例，有些配置是强制性的。特别地，如果没有用户指定的SECRET_KEY值，Superset将不会启动。请参考[配置Superset](https://superset.apache.org/docs/configuration/configuring-superset)

上面爆不安全的SECRET_KEY也是之前找问题输入了一句export SUPERSET_SECRET_KEY=“123456”

而找到~/superset/lib/pytohn3.8/site-packages/superset目录下的config.py配置文件可以发现，配置里就是SECRET_KEY调环境变量的SUPERSET_SECRET_KEY

文档里面的注释建议用openssl rand -base64 42那就在终端中用该命令生成一个随机密钥再赋值给SUPERSET_SECRET_KEY

  1. 执行superset db upgrade命令,遇到模块缺失问题就pip install安装，或者重装

  2. 最后superset db upgrade命令执行成功

  3. 接着根据官网文档依次执行下面的命令

    export FLASK_APP=superset命令上面执行过了

输入superset fab create-admin创建管理员用户，不输入默认就是[ ]中的内容，密码输入不可见

输错了CTRL+C中断重新执行该命令

superset load_examples下载一些样例数据，应该是要开代理连接的

superset init创建默认角色和权限

superset run -p 8088 –with-threads –reload –debugger 在端口 8088 上启动 Web 服务器，使用 -p 绑定端口

  1. 打开浏览器，网址栏输入localhost:8088回车就有了，输入之前创建的管理员用户[admin]和密码
成功进入（因为没改代理跳过了superset load_examples一步所以主菜单没显示样例数据）

  1. 到此，apache-superset:2.1部署完成

    其他配置与问题可查阅参考资料
